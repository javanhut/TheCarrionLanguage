Syntax and Terminology

Carrion uses mystical terminology inspired by magic and Norse mythology. This guide explains the language's unique vocabulary and Python-inspired syntax.

Magical Terminology

Carrion Term Traditional Term Description
spell function A reusable block of code that performs a specific task
grim class A grimoire (spellbook) that defines a blueprint for objects
init constructor The initialization spell called when creating a new grimoire instance
attempt try Begin a block that may encounter errors
ensnare catch/except Capture and handle an error
resolve finally Always execute cleanup code
repeat for i in range(n) Execute a block a specific number of times
Munin standard library Named after Odin's raven of memory

Basic Syntax

Comments

// Single-line comment

/*
   Multi-line
   comment
*/

Indentation

Carrion uses indentation (4 spaces recommended) to define code blocks:

spell greet(name):
    message = f"Hello, {name}!"    // Inside the spell
    print(message)                  // Still inside

print("Outside the spell")          // Outside
Important: Use consistent indentation throughout your code. Mixing tabs and spaces will cause errors.

Statements

Statements end at the end of the line (no semicolons needed):

x = 10
y = 20
result = x + y

Defining Spells (Functions)

// Basic spell
spell greet():
    print("Hello, World!")

// Spell with parameters
spell add(a, b):
    return a + b

// Spell with default parameter
spell greet(name="Stranger"):
    print(f"Hello, {name}!")

// Call spells
greet()              // Hello, World!
result = add(5, 3)   // 8
greet("Mage")        // Hello, Mage!

Defining Grimoires (Classes)

grim MagicalCreature:
    // Constructor
    init(name, power):
        self.name = name
        self.power = power

    // Method (spell)
    spell describe():
        return f"{self.name} has power level {self.power}"

    // Method with parameters
    spell attack(target):
        print(f"{self.name} attacks {target}!")

// Create an instance
dragon = MagicalCreature("Fafnir", 100)
print(dragon.describe())
dragon.attack("the castle")

Control Flow Syntax

If/Elif/Else

if condition:
    // code when true
elif other_condition:
    // code when elif is true
else:
    // code when all conditions are false

For Loops

// Iterate over a list
for item in items:
    print(item)

// Iterate with index using range
for i in range(5):
    print(i)  // 0, 1, 2, 3, 4

// Range with start and end
for i in range(1, 10):
    print(i)  // 1 through 9

While Loops

count = 0
while count < 5:
    print(count)
    count = count + 1

Repeat Loops

// Carrion's unique repeat construct
repeat 5:
    print("Hello!")  // Prints 5 times

String Formatting

name = "Carrion"
version = 0.19

// F-strings (interpolation)
message = f"Welcome to {name} version {version}"

// Concatenation
greeting = "Hello, " + name + "!"

// Multi-line strings
story = """
Once upon a time,
in a land of magic,
there lived a crow...
"""

Operators

Arithmetic

a + b    // Addition
a - b    // Subtraction
a * b    // Multiplication
a / b    // Division
a % b    // Modulo (remainder)
a ** b   // Exponentiation

Comparison

a == b   // Equal
a != b   // Not equal
a < b    // Less than
a > b    // Greater than
a <= b   // Less than or equal
a >= b   // Greater than or equal

Logical

a and b  // Both true
a or b   // Either true
not a    // Negation
x in y   // Membership test

Error Handling Syntax

attempt:
    // Code that might fail
    result = risky_operation()
ensnare SpecificError as e:
    // Handle specific error
    print(f"Caught: {e}")
ensnare:
    // Handle any other error
    print("Something went wrong")
resolve:
    // Always runs (cleanup)
    cleanup()

Import Syntax

// Import a module
import "mymodule"

// Import specific item
import "mymodule.MyGrimoire"
import "mymodule.my_spell"

// Use imported items
mymodule.function()
obj = MyGrimoire()
