Error Reference

Carrion v0.1.9 features an enhanced error system with intelligent suggestions and detailed context. This guide explains common errors and how to fix them.

Syntax Errors

Unexpected Token E0001

The parser encountered a token it wasn't expecting at that location.

// Error
spell greet(name)    // Missing colon
    print(name)

// Correct
spell greet(name):
    print(name)
Tip: Check for missing colons after spell/grim/if/for/while declarations.

Indentation Error E0002

Inconsistent indentation in a code block.

// Error
spell example():
    x = 1
  y = 2    // Inconsistent indentation

// Correct
spell example():
    x = 1
    y = 2
Tip: Use consistent 4-space indentation throughout your code.

Type Errors

Type Mismatch E0101

An operation was attempted on incompatible types.

// Error
result = "42" + 10   // String + Integer

// Error message (v0.1.9):
// error: String/Number type mismatch
// help:
//   - Convert string to number: int("42")
//   - Convert number to string: str(10)

// Correct
result = int("42") + 10   // 52
result = "42" + str(10)   // "4210"

Unsupported Operand Types E0102

An operator doesn't support the given operand types.

// Error
result = [1, 2] - [3]   // Lists don't support subtraction

// Correct - use appropriate list operations
result = [1, 2]
result.remove(1)

Name Errors

Identifier Not Found E0201

A variable or function name was used that hasn't been defined.

// Error
name = "Carrion"
print(nme)   // Typo!

// Error message (v0.1.9):
// error: identifier not found: nme
//   --> example.crl:2:7
// help: Did you mean 'name'?

// Correct
print(name)
New in v0.1.9: The error system now suggests similar variable names when you make typos!

Undefined Method E0202

A method was called that doesn't exist on the object.

// Error
text = "hello"
text.uppercase()   // Method doesn't exist

// Error message (v0.1.9):
// error: undefined method 'uppercase' for String
// help: Did you mean 'upper'?

// Correct
text.upper()

Runtime Errors

Index Out of Bounds E0301

Attempted to access an index that doesn't exist.

// Error
items = [1, 2, 3]
print(items[10])   // Only indices 0-2 exist

// Error message:
// error: index out of bounds: 10 (length: 3)

// Correct
if len(items) > 10:
    print(items[10])

Key Not Found E0302

Attempted to access a hash key that doesn't exist.

// Error
data = {"name": "Carrion"}
print(data["version"])   // Key doesn't exist

// Correct - check first or use get with default
if "version" in data:
    print(data["version"])
else:
    print("No version")

Division by Zero E0303

Attempted to divide by zero.

// Error
result = 10 / 0

// Correct
divisor = get_divisor()
if divisor != 0:
    result = 10 / divisor
else:
    print("Cannot divide by zero")

Import Errors

Module Not Found E0401

The specified module could not be found.

// Error
import "nonexistent_module"

// Correct - ensure the module exists in your path
import "existing_module"
Tip: Check that the module file exists and the path is correct.

Error Handling Best Practices

// Always handle potential errors
attempt:
    result = risky_operation()
ensnare ValueError as e:
    print(f"Invalid value: {e}")
ensnare IOError as e:
    print(f"File error: {e}")
ensnare:
    print("An unexpected error occurred")
resolve:
    cleanup_resources()

Understanding Error Messages

Carrion v0.1.9 provides detailed error messages with:

  • Location: File name, line number, and column
  • Context: The relevant source code line
  • Pointer: Caret indicating the exact error position
  • Suggestion: "Did you mean?" recommendations
  • Help: Guidance on how to fix the error
error: identifier not found: prnt
  --> example.crl:5:1
   |
 5 | prnt("Hello")
   | ^^^^ unknown identifier
   |
help: Did you mean 'print'?
