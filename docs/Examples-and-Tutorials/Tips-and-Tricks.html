Tips and Tricks

Master the art of Carrion programming with these expert tips and techniques.

REPL Tips

1Use Mimir for Help

Access the built-in documentation system for instant help:

# In REPL or terminal
mimir                    # Interactive mode
mimir scry print         # Get help for specific function
mimir list               # List all functions

2Tab Completion

Use Tab for autocompletion of keywords, function names, and variables in the REPL.

>>> pri<TAB>
print

>>> my_var.upp<TAB>
my_var.upper()

3Multiline Input

The REPL automatically handles multiline input for blocks:

>>> spell greet(name):
...     return f"Hello, {name}!"
...
>>> greet("World")
Hello, World!

Code Organization Tips

4Use Meaningful Names

Carrion's magical terminology encourages descriptive naming:

// Good - descriptive names
grim UserAuthentication:
    spell validate_credentials(username, password):
        ...

// Avoid - cryptic names
grim UA:
    spell vc(u, p):
        ...

5Organize with Modules

Split large projects into logical modules:

// utils.crl
spell format_date(date):
    ...

spell validate_email(email):
    ...

// main.crl
import "utils"

email = input("Enter email: ")
if utils.validate_email(email):
    print("Valid!")

6Use Selective Imports

Import only what you need:

// Import specific grimoire
import "module.SpecificGrimoire"

// Import specific spell
import "utils.helper_function"

Performance Tips

7Cache Computed Values

Avoid recalculating values in loops:

// Slow - len() called every iteration
for i in range(len(items)):
    process(items[i])

// Fast - len() called once
count = len(items)
for i in range(count):
    process(items[i])

8Use Appropriate Data Structures

Choose the right data structure for your needs:

// Hash for fast lookups
user_cache = {}
user_cache[user_id] = user_data

// List for ordered collections
queue = []
queue = append(queue, item)

// BTree for sorted data
sorted_data = BTree()
sorted_data.insert(value)

Error Handling Tips

9Be Specific with Errors

Catch specific error types when possible:

attempt:
    data = File.read(filename)
    parsed = parse_json(data)
ensnare FileNotFoundError:
    print(f"File not found: {filename}")
ensnare ParseError as e:
    print(f"Invalid JSON: {e}")
ensnare:
    print("Unexpected error occurred")

10Use resolve for Cleanup

Always clean up resources in the resolve block:

file = None
attempt:
    file = File.open("data.txt", "r")
    content = file.read()
    process(content)
ensnare:
    print("Error processing file")
resolve:
    if file != None:
        file.close()

Testing Tips

11Use Sindri for Testing

Write tests using the Sindri testing framework:

// test_math.crl
spell appraise_addition():
    check(2 + 2 == 4)
    check(add(5, 3), 8)

spell appraise_division():
    check(10 / 2 == 5)
    check(divide(9, 3), 3)

// Run with: sindri appraise test_math.crl

12Test Edge Cases

Always test boundary conditions:

spell appraise_edge_cases():
    // Empty inputs
    check(len("") == 0)
    check(len([]) == 0)

    // Negative numbers
    check(abs(-5), 5)

    // Zero division handled
    check(safe_divide(10, 0), None)

Common Gotchas

!String vs Integer Comparison

Be careful when comparing user input:

// input() returns a string!
age = input("Enter age: ")

// Wrong - comparing string to int
if age > 18:  // Error!
    print("Adult")

// Correct - convert first
if int(age) > 18:
    print("Adult")

!Mutable Default Arguments

Avoid using mutable objects as default arguments:

// Problematic - shared list
spell add_item(item, items=[]):
    items = append(items, item)
    return items

// Better - use None as default
spell add_item(item, items=None):
    if items == None:
        items = []
    items = append(items, item)
    return items

!BTree Method Name Change

In v0.1.9, size() was renamed to get_size():

// Old (v0.1.8)
count = my_tree.size()

// New (v0.1.9+)
count = my_tree.get_size()
